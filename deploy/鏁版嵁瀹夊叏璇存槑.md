# E23跑起来 · 数据安全说明

## 一、数据分类与存储位置

| 数据 | 存储位置 | 说明 |
|---|---|---|
| 账号（手机号/昵称/密码哈希） | 服务端 PostgreSQL | 密码 bcrypt(10) 哈希，绝不存明文 |
| 运动记录与轨迹点 | 服务端 PostgreSQL + 本机缓存 | 本机缓存用于离线，服务端为唯一事实源 |
| 登录凭证（JWT） | 本机 localStorage | 30 天过期；退出登录即清除 |
| 班级邀请码 | 服务端 | 限次数、可设过期 |
| 审计日志 | 服务端 audit_logs | 注册/审批/活动裁决/删除/进度重算全留痕 |

## 二、传输与访问控制

- 全链路 HTTPS（Nginx 终结 TLS），安卓/iOS 不允许明文 HTTP；
- JWT 无状态认证，接口级鉴权：未登录 401、未审批 403（NOT_APPROVED）、非管理员 403；
- 手机号在所有对外接口中脱敏返回（`139****2222`），排行榜只暴露昵称与头像；
- `pending` 成员只读：可登录浏览，不可贡献跑量，不可进入排行榜。

## 三、数据可信机制（防造假）

1. **服务端校验引擎**：距离/时长/配速/时间一致性硬校验，超人类极限直接 rejected；
2. **轨迹质量分析**：GPS 跳变（>500m 且速度>8m/s）占比>5% → pending；精度>60m 的点过半 → pending；
3. **手动补录一律 pending**：必须填写凭证说明，管理员审核通过才计入；单人单日手动上限 42.2km；
4. **幂等防重**：clientId 唯一约束，重试/重放不产生重复里程；
5. **汇总重算制**：任何增删改后 route_progress 立即按「仅 valid + 仅 approved」重算，无手工改数空间；
6. **审计日志**：每一次状态变更记录操作者、对象与时间，可追溯。

## 四、隐私边界（当前版本如实声明）

- 轨迹坐标仅用于校验与展示，不做地理围栏外的二次利用；
- 本机缓存（localStorage）包含跑步记录与 token，共用设备请退出登录；
- 头像图片以 DataURL 存本机（预览版），正式版将改为服务端对象存储；
- 未接入任何第三方统计/埋点 SDK，无数据外发；
- 微信授权登录尚未接入；当前账号体系为过渡方案，接入微信后手机号数据将做迁移与去重。

## 五、备份与恢复

- 每日 `pg_dump` 备份 + 异地副本（见《云端部署说明》第六节）；
- 灾难恢复：重装 → restore → 前端无需变更（本机缓存失效后重新登录即可，服务端记录完整）。
