# E23跑起来 · 云端部署说明

> 适用对象：运维 / 后端同学。目标：把「前端 PWA + 后端 API + PostgreSQL」跑在一台云主机上。
> 最低配置：1 核 2G（演示）/ 2 核 4G（班级正式）。系统：Ubuntu 22.04+，Docker 24+。

## 一、架构

```
手机浏览器 / 安卓 APK ──HTTPS──▶ Nginx ──▶ 前端静态文件（dist/）
                                  └─▶ /api/* ──▶ Fastify 后端 :8787 ──▶ PostgreSQL :5432
```

## 二、后端一键启动（Docker Compose）

```bash
cd backend
cp ../deploy/.env.example .env      # 按模板填写，JWT_SECRET 必须更换
docker compose up -d --build        # 启动 postgres + backend
docker compose exec backend npm run seed   # 首次：写入路线版本/管理员/邀请码
docker compose logs -f backend      # 观察启动日志
curl http://127.0.0.1:8787/api/health
# 期望返回：{"ok":true,"service":"e23-backend","db":"up","serverTime":"..."}
```

compose 启动命令内部先执行 `prisma migrate deploy || prisma db push --skip-generate` 建表，再启动服务。

## 三、前端构建与 Nginx

```bash
# 构建时注入后端地址与提交号
VITE_API_BASE_URL=https://api.你的域名.com \
VITE_COMMIT_SHA=$(git rev-parse HEAD) \
npm ci && npm run build
```

Nginx 参考配置（`/etc/nginx/sites-available/e23`）：

```nginx
server {
    listen 443 ssl;
    server_name 你的域名.com;
    root /var/www/e23/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;   # SPA 回退
    }
    location /api/ {
        proxy_pass http://127.0.0.1:8787;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

HTTPS 证书推荐 `certbot --nginx`。安卓 WebView / iOS PWA 都要求后端必须 HTTPS，否则混合内容被拦截。

## 四、本地开发（不用 Docker）

```bash
# 根目录一键装依赖
npm run setup
# 启动数据库（任选其一）：
#   A. Docker: docker compose -f backend/docker-compose.yml up -d postgres
#   B. 本机已有 PostgreSQL：建库 e23，导出 DATABASE_URL
cd backend && cp ../deploy/.env.example .env
npm run dev        # tsx watch，默认 :8787
npm run seed       # 首次初始化
# 前端另开终端
cd .. && VITE_API_BASE_URL=http://localhost:8787 npm run dev
```

## 五、管理员日常工作

| 事项 | 方法 |
|---|---|
| 生成邀请码 | `POST /api/admin/invites`（管理员 token） |
| 审批新成员 | `POST /api/admin/members/:id/review` |
| 审核手动补录 | `GET /api/admin/activities?status=pending` → `POST /api/admin/activities/:id/review` |
| 删除造假记录 | `DELETE /api/admin/activities/:id`（汇总自动重算） |

## 六、备份

```bash
docker compose exec postgres pg_dump -U e23 e23 > backup-$(date +%F).sql
```

建议每日 cron + 异地副本。恢复：`psql -U e23 e23 < backup.sql`。
