# 风险清单

## Phase 1.4 Codex GPS Map Fix 补充（2026-07-22）

- Android 原生 GPS 状态机、SQLite 审计字段与真实个人轨迹地图已进入代码验证阶段；真机实走、锁屏 5 分钟和厂商后台限制仍必须按独立验收清单验证。
- 本机 Android 构建环境无法从 Maven 下载 Android Gradle Plugin 8.7.2 与 Google Services Plugin 4.4.2；以专用 GitHub Actions 构建结果为准。
- 跑步页开发测试底图使用 OSM 在线瓦片，不做预取或离线抓取；中国大陆正式发布前需完成地图服务、Key、资质、隐私和合规决策。
- 当前仍为预览测试版：不启用真实微信、生产数据库、正式多人和正式路线，V2路线保持 `DRAFT`。

## P0（阻断正式使用）

1. **无公网部署**：当前仅在对话平台内预览，无 Safari 直开的公网 HTTPS 地址 → 第二阶段必须完成公开部署
2. **代码未推送 GitHub**：本地仓库无远程，SHA 无法在 GitHub 核验 → 需推送到 kimi/e23-v2-product 分支
3. **无 APK**：本环境无 Android SDK，未构建 APK；android/ 工程未包含在本仓库 → 由 Codex 在第二阶段按 README 步骤从同一 Commit 构建
4. **本机数据孤岛**：跑步记录仅存 localStorage，换设备/清缓存即丢失；多人后端上线前无法避免 → 需在发布说明中告知用户

## P1（影响体验）

5. **路线里程 DRAFT**：分段里程非实测，估计误差 ±10% → 第四阶段实测核验
6. **iPhone 未真机测试**：目前为 iPhone 14 视口等效环境（Chromium）测试，Safari 特有行为（定位授权弹窗、主屏幕全屏）需真机复核
7. **GPS 仅前台**：浏览器锁屏/切后台后定位中断，长距离跑步记录可能断点 → 第三阶段原生化解决
8. **主包 389KB**：含版图轮廓与路线数据，可接受；后续若加高德 SDK 需重新评估体积

## P2（待办不阻断）

9. 站点坐标为城镇中心近似值
10. 海峡轮渡段未单列
11. 藏南段以官方公开版图为准，需在发布材料中保持同一口径
12. 测试登录入口在正式上线前必须移除或加环境开关（当前默认开启，正式构建应设 `VITE_TEST_SMS_CODE` 为空并关闭入口）

## 数据合规

13. 不提交密钥/Token/真实服务账号（已遵守）
14. 正式微信登录涉及用户头像/昵称，需隐私政策与授权弹窗（第三阶段）
