## 三、已完成代码（事实清单）

**后端**
1. 统一适配器契约 `backend/src/services/providers.ts`：12 方法（getAuthorizationUrl / exchangeAuthorization / refreshAuthorization / revokeAuthorization / pullActivities / normalizeActivity / getSyncCursor / saveSyncCursor / handleWebhook / verifyWebhookSignature / mapProviderError + enabled）；**平台不支持的方法一律显式返回 `not_supported`，无空实现**（如佳明无 refresh、悦跑圈/佳明无公开撤销端点、三平台 Webhook 均未接入）。
2. 安全层：
   - 令牌 AES-256-GCM 加密落库（`services/crypto.ts`，密钥 `TOKEN_ENCRYPTION_KEY` 派生，生产缺省拒绝回退）；DB 中无明文 access/refresh token（测试 24 验证）
   - OAuth state：一次性（OAuthState 表，重用→400「已被使用（防重放）」）、10 分钟短时效、JWT 绑定用户与平台
   - 回调地址白名单校验（`CALLBACK_ALLOWED_HOSTS`）
   - 断开连接：调用平台撤销（华为已实现调用；悦跑圈/佳明无公开端点如实返回 not_supported）→ 本地凭据清除 → state 清除
   - 审计仅记录脱敏 openId（`maskId`）；日志不打印令牌
   - Webhook 接收端：验签未接入的平台一律 501 NOT_SUPPORTED
3. 真实状态接口 `GET /api/v1/integrations/catalog`：每平台返回 provider / display_name / connection_type / implementation_status / credential_status / sandbox_status / production_status / supported_activity_types / required_qualifications / privacy_requirements / commercial_risk / user_visible_message（佳明另附 product_facts）。
4. 管理员状态推进端点 `POST /api/admin/integrations/:provider/mark`：凭真实事件推进状态并留痕。
5. 微信小程序数据提交预留接口 `POST /api/v1/integrations/wechat-miniprogram/activities`：未配置密钥一律 503；启用后记录一律 pending 人工审核。

**前端**
6. 按钮渲染**只认服务端 catalog 状态**：`mock_verified` 及以下 → 「尚未开放」（禁用，附资质与商业风险说明）；`sandbox_connected` → 「测试连接」；`production_connected` 及以上 → 「授权连接」。**不因配置文件存在而显示「授权连接」**。
7. 微信/Apple 行无任何授权按钮，如实说明前置条件（小程序 / iOS 原生）。
8. 悦跑圈 GPX/TCX 轨迹导入与凭证补录保留（该两条不依赖官方凭据，真实可用）。

## 八、隐私与合规风险

- 运动健康数据属敏感个人信息：需用户明示授权、隐私政策覆盖第三方数据用途、提供断开与删除通道；
- 令牌必须加密存储（已实现 AES-256-GCM）；日志/审计不得含令牌与完整隐私数据（已实现脱敏）；
- Apple HealthKit：App Store 审核对健康数据用途有硬性要求，禁止上传原始健康数据用于广告等用途；
- 华为 Health Kit：权限审核较严，非医疗/非企业用途可能受限；
- 佳明：开发者协议含数据使用限制与缓存期限要求；
- 微信小程序：需单独隐私协议，步数数据经小程序服务端中转需加密。
