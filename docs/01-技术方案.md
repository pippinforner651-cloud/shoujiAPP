# 全民环游中国虚拟跑步地图 — 技术方案 V2

> 编写日期：2026-07-08 | 状态：已确认（React + Vite 路线）
> 修订依据：总控审核 V1 → V2，确定采用 React + Vite 组件化架构

---

## 一、产品概述

一款将现实跑步数据映射到虚拟中国地图上的 Web 应用。用户记录每日跑步里程，累积后在地图上沿「中国环游经典自驾路线」推进，打卡 48 个关键城市节点，解锁成就徽章。

**MVP 核心交互**：输入跑步里程 → 地图上角色沿路线前进 → 抵达城市 → 弹出打卡卡片（城市名+省份+景点介绍）。

---

## 二、技术选型（最终确定）

| 层面 | 选型 | 理由 |
|------|------|------|
| 框架 | **React 18 + Vite** | 组件化清晰、生态成熟、Vite 构建快 |
| 语言 | TypeScript | 类型安全，降低后期重构成本 |
| 地图 | **ECharts 中国地图**（echarts + echarts/map/js/china.js） | 轻量、无 CDN 跨域问题、API 友好、散点+连线原生支持 |
| 样式 | **Tailwind CSS v3** | 快速响应式布局，暗色模式支持 |
| 状态 | **Zustand** | 轻量（~1KB），比 Redux 更适合中小项目 |
| 路由 | React Router v6 | 标准 SPA 路由 |
| 数据存储 | **localStorage**（MVP）→ IndexedDB（V2）→ 后端数据库（V3） | 渐进式升级 |
| 构建 | Vite | 开发 HMR 快，生产构建优 |
| 部署 | CloudStudio 静态站点 | 公网可达，免费 |

### 备选对比（已否决）

| 方案 | 状态 | 否决原因 |
|------|------|----------|
| 零依赖单文件 HTML | ❌ 已否决 | 无法支撑多用户、排行榜、组件化扩展 |

## 四、关键技术决策

### 4.1 地图方案：ECharts 中国地图

选择 **ECharts** 而非 Leaflet/Mapbox：
- **零 CDN 依赖**：echarts 和 china.js 地图数据打包进 bundle，不依赖外部 CDN
- **视觉效果丰富**：散点图、飞线图、热力图原生支持
- **中国地图合规**：ECharts 官方 china.js 已正确标注「中国香港」「中国澳门」「中国台湾」
- **移动端友好**：内置触屏手势缩放和平移

数据来源：echarts 官方 `china.js` 地图数据（含 34 个省级行政区 GeoJSON）。

### 4.2 路线数据设计

- 名称：「中国环游经典自驾路线」
- 总长：约 **21,800 公里**（虚拟地图里程）
- 节点数：**48 个城市**
- 比例：**跑步 1 km = 虚拟路线 10 km**
- 完成全程约需跑步 **2,180 km**

路线走向：
```
北京 → 承德 → 沈阳 → 长春 → 哈尔滨 → 呼伦贝尔 → 呼和浩特 → 包头 → 
银川 → 兰州 → 西宁 → 嘉峪关 → 敦煌 → 吐鲁番 → 乌鲁木齐 → 库尔勒 → 
阿克苏 → 喀什 → 阿里 → 日喀则 → 拉萨 → 林芝 → 香格里拉 → 丽江 → 
大理 → 昆明 → 贵阳 → 桂林 → 南宁 → 北海 → 湛江 → 海口 → 三亚 → 
深圳 → 厦门 → 福州 → 温州 → 杭州 → 上海 → 南京 → 合肥 → 武汉 → 
郑州 → 西安 → 太原 → 石家庄 → 天津 → 北京（环线闭合）
```

路线数据以 TypeScript 常量形式定义在 `src/data/route.ts`，节点结构：
```typescript
interface RouteNode {
  id: string;           // 城市英文ID
  name: string;          // 城市中文名
  province: string;      // 省份
  coordinates: [number, number]; // [经度, 纬度]
  distanceFromStart: number;     // 距起点累计公里
  landmark: string;      // 代表景点
  description: string;   // 景点简介
}
```

### 4.3 马拉松里程碑体系

在 48 个城市之间穿插 **9 个马拉松里程碑**，降低用户正向反馈间隔：

| 序号 | 累积跑步 | 对应虚拟里程 | 里程碑名称 |
|------|----------|-------------|-----------|
| 1 | 5 km | 50 km | 迷你马 |
| 2 | 10 km | 100 km | 十分马拉松 |
| 3 | 21.0975 km | 211 km | 🏅 半程马拉松 |
| 4 | 42.195 km | 422 km | 🏆 全程马拉松 |
| 5 | 100 km | 1,000 km | 百公里勇士 |
| 6 | 250 km | 2,500 km | 超级马拉松 |
| 7 | 500 km | 5,000 km | 千里征途 |
| 8 | 1000 km | 10,000 km | 万里长征 |
| 9 | 2000 km | 20,000 km | 环游中国 |

## 六、风险与约束

| 风险 | 影响 | 对策 |
|------|------|------|
| npm install 沙箱拦截 | 阻塞 | 参考踩坑日志，必要时用 cnpm 或离线依赖包 |
| ECharts china.js 地图合规 | 合规 | 使用 ECharts 官方 china.js，自带正确标注 |
| localStorage 5MB 限制 | 远期 | MVP 预估 < 2MB，够用；V2 切 IndexedDB |
| 移动端 ECharts 触屏 | 体验 | ECharts 原生支持手势，实测验收 |
| 路线距离精度 | 用户体验 | 明确标注"估算高速公路距离"，非精确值 |
