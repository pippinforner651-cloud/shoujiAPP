# 用户账户与云同步设计 V1

> **项目**：全民环游中国虚拟跑步地图  
> **版本**：V2.0 Phase 3.1  
> **日期**：2026-07-09

---

## 一、设计目标

将当前 localStorage 单机模式升级为支持多人在线的数据架构，为未来接入真实后端做准备。

---

## 二、用户模型

### UserAccount

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `string` | 用户唯一 ID（游客自动生成 `guest_xxx`） |
| `nickname` | `string` | 昵称 |
| `avatar` | `AvatarType` | 头像类型 |
| `createdAt` | `string` | 账户创建时间 |
| `isGuest` | `boolean` | 是否为游客模式 |
| `level` | `number?` | 等级（同步自成就会） |
| `experience` | `number?` | 总经验值 |
| `lastSyncAt` | `string?` | 上次同步时间 |
| `cloudUserId` | `string?` | 云端账号 ID（登录后关联） |

### 游客 vs 登录用户

| 维度 | 游客 | 登录用户 |
|------|------|---------|
| 用户 ID | 自动生成 `guest_xxx` | 云端注册 `cloud_xxx` |
| 数据存储 | localStorage | localStorage + 云端 |
| 多设备同步 | ❌ | ✅ |
| 排行榜参与 | ✅ | ✅ |
| 数据恢复 | ❌ 换设备丢失 | ✅ |

## 三、数据同步架构

```
┌─────────────┐     ┌──────────────┐     ┌───────────┐
│  localStorage │ ←──→ │  syncService  │ ←──→ │  Cloud API │
│  (主存储)     │     │  (同步逻辑)   │     │  (Mock)    │
└─────────────┘     └──────────────┘     └───────────┘
       ↑                    ↑                    ↑
   runStore           fullSync()           userService
   userStore                               activityService
   achievementStore
```

### 同步流程

```
用户触发同步（手动/定时）
  ↓
syncService.fullSync()
  ├── 1. uploadProfile()      ← 用户档案
  ├── 2. uploadActivities()   ← 跑步记录（增量）
  ├── 3. downloadActivities() ← 拉取云端记录
  ├── 4. mergeActivities()    ← 合并（云端优先）
  ├── 5. 写回 localStorage
  ├── 6. 刷新成就
  └── 7. 更新 lastSyncAt
```

### 合并策略

```typescript
function mergeActivities(local, cloud): merged[] {
  // 1. 以 record.id 为 key 建立 Map
  // 2. 本地记录先插入
  // 3. 云端记录覆盖（云端数据较新）
  // 4. 按日期 + 创建时间排序
}
```

## 四、API 接口设计

### userService

| 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|
| `registerUser` | nickname | CloudUser | 注册新用户 |
| `bindCloudAccount` | guestId, nickname | CloudUser | 游客转正 |
| `fetchUser` | cloudUserId | CloudUser|null | 获取用户信息 |
| `uploadProfile` | UserAccount | boolean | 上传用户档案 |

### activityService

| 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|
| `uploadActivities` | userId, records[] | {success, uploaded} | 上传记录 |
| `downloadActivities` | userId, sinceDate? | records[] | 下载记录 |
| `mergeActivities` | local[], cloud[] | merged[] | 合并数据 |

### syncService

| 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|
| `fullSync` | 无 | SyncResult | 全量同步 |
| `shouldSync` | 无 | boolean | 检查是否需要同步 |

## 六、Mock 云服务

### 当前实现

所有 `services/cloud/` 下的模块均为 Mock 实现：
- `userService.ts` — 模拟注册/登录，`console.log` 记录
- `activityService.ts` — 内存数组存储，去重合并
- `syncService.ts` — 编排同步流程，调用 cloud / 回写 localStorage

### 接入真实后端

替换 `services/cloud/` 中的实现即可：

```typescript
// 伪代码示例
export async function uploadActivities(records) {
  const token = await getAuthToken();
  const res = await fetch('https://api.example.com/v1/activities', {
    method: 'POST',
    headers: { Authorization: `Bearer ${token}` },
    body: JSON.stringify({ records }),
  });
  return res.json();
}
```
