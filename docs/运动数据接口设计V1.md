# 运动数据接口设计 V1

> **项目**：全民环游中国虚拟跑步地图  
> **版本**：V2.0 Phase 2.1  
> **日期**：2026-07-09

---

## 一、设计目标

建立统一运动数据模型，为未来接入手机 APP、运动手表、第三方平台准备。

### 数据流架构

```
Apple Watch ─┐
华为手表 ────┤
Garmin ─────┤              activityAdapter              runStore
小米运动 ────┤  External ───→  (统一适配层) ───→  ActivityRecord ─→  RunRecord
微信运动 ────┤     Input                                          ↓
GPS/手动 ───┘                                              localStorage
                                                              ↓
                                                       progressStore
                                                              ↓
                                                     globalProgressStore
```

### 分层说明

| 层 | 职责 | 文件 |
|----|------|------|
| 数据模型 | 定义统一 ActivityRecord | `types/activity.ts` |
| 适配器 | 外部格式 → 统一格式 | `services/activityAdapter/` |
| 存储 | 持久化到 localStorage | `store/runStore.ts` |
| 消费 | 个人/全民进度计算 | `store/progressStore.ts`, `store/globalProgressStore.ts` |

## 二、数据模型

### ActivityRecord（统一运动记录）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | `string` | ✅ | 唯一 ID（自动生成） |
| `userId` | `string` | ✅ | 用户标识 |
| `source` | `ActivitySource` | ✅ | 数据来源 |
| `device` | `DeviceInfo` | ❌ | 设备信息 |
| `activityType` | `ActivityType` | ✅ | 运动类型 |
| `distanceKm` | `number` | ✅ | 距离（公里） |
| `durationSec` | `number` | ✅ | 时长（秒） |
| `paceSec` | `number` | ✅ | 配速（秒/公里，自动计算） |
| `calories` | `number` | ❌ | 消耗热量（千卡） |
| `startTime` | `string` | ✅ | 运动开始时间（ISO 8601） |
| `gpsTrack` | `GpsTrack` | ❌ | GPS 轨迹 |
| `createdAt` | `string` | ✅ | 记录创建时间 |
| `note` | `string` | ❌ | 备注 |

### ActivitySource 枚举

| 值 | 来源 | 适配器 |
|----|------|--------|
| `manual` | 手动录入 | `manualAdapter.ts` |
| `apple_health` | Apple Watch / Apple Health | `mockAppleAdapter.ts` |
| `huawei_health` | 华为运动健康 | `mockAppleAdapter.ts` |
| `garmin` | Garmin | 预留 |
| `xiaomi` | 小米运动 | 预留 |
| `wechat` | 微信运动 | 预留 |
| `gps` | GPS 设备 | 预留 |

### GpsPoint（GPS 轨迹点）

| 字段 | 类型 | 说明 |
|------|------|------|
| `latitude` | `number` | 纬度 |
| `longitude` | `number` | 经度 |
| `timestamp` | `string` | 时间戳 |
| `altitude` | `number` | 海拔（可选） |

## 三、适配器层

### 目录结构

```
src/services/activityAdapter/
├── index.ts              ← 统一入口：adaptToRunStore / adaptMultipleToRunStore
├── types.ts              ← 类型 + 标签映射
├── manualAdapter.ts      ← 手动录入适配器
└── mockAppleAdapter.ts   ← Apple Watch / 华为手表模拟适配器
```

### 接口

```typescript
// 统一输入格式
interface ExternalActivityInput {
  source: ActivitySource;
  device?: DeviceInfo;
  activityType: ActivityType;
  distanceKm: number;
  durationSec: number;
  calories?: number;
  startTime: string;
  gpsTrack?: GpsTrack;
  note?: string;
}

// 适配结果
interface AdaptResult {
  success: boolean;
  recordId?: string;
  error?: string;
  virtualKm?: number;
}

// 核心函数
function adaptToRunStore(input: ExternalActivityInput): AdaptResult;
function adaptMultipleToRunStore(inputs: ExternalActivityInput[]): AdaptResult[];
```

### 添加新适配器

```typescript
// 步骤 1：创建适配函数
function parseGarminFit(data: ArrayBuffer): ExternalActivityInput { ... }

// 步骤 2：调用统一入口
const result = adaptToRunStore(parseGarminFit(fitData));
```

## 四、存储层

### RunRecord（存储格式）

向后兼容旧版 `RunRecord`，增加扩展字段：

```typescript
interface RunRecord {
  id: string;           // 唯一 ID
  date: string;         // 日期 YYYY-MM-DD
  distanceKm: number;   // 公里
  durationMin: number;  // 分钟
  pace: number;         // 配速 分钟/公里
  createdAt: string;    // 创建时间
  note?: string;        // 备注
  // V2.1 扩展字段：
  source?: ActivitySource;
  device?: DeviceInfo;
  activityType?: ActivityType;
  calories?: number;
}
```

### localStorage 版本迁移

| 版本 | 说明 |
|------|------|
| v1 | 基础字段（Phase 2.1 之前） |
| v2 | 增加 source/device/activityType/calories |

迁移策略：`STORAGE_VERSION` 从 1 升到 2，旧数据自动补 `source: 'manual'`。

## 六、未来 API 接入方案

```
iPhone App
  → HealthKit API
    → 读取 HKWorkout
      → 构建 ExternalActivityInput
        → adaptToRunStore()

```
华为手机 App
  → 华为运动健康 SDK

## 七、验证方法

### 测试场景 1：手动输入 5km

1. 在跑步输入表单输入 5km、30min
2. 点击 "+ 添加"
3. ✅ 记录添加成功，累计跑量 +5km

### 测试场景 2：模拟 Apple Watch 8.2km

1. 在"今日运动"卡片点击 `🍎 模拟 Apple Watch（8.2 km）`
2. ✅ 显示"✅ Apple Watch 同步成功：8.2 km"
3. ✅ 今日统计更新：距离 +8.2km
4. ✅ 累计跑量 +8.2km

### 测试场景 3：模拟华为手表 6.5km

1. 点击 `📱 模拟华为手表（6.5 km）`
2. ✅ 显示"✅ 华为手表同步成功：6.5 km"
3. ✅ 今日统计更新
4. ✅ 三者累计 = 5 + 8.2 + 6.5 = 19.7km

### 数据一致性校验

```
手动 5km + Apple Watch 8.2km + 华为 6.5km
  = ActivityRecord 累计 19.7km
    = 个人累计跑量 19.7km
      = 全民累计贡献（通过 globalProgressStore 导出后求和）
```
