# 全民环游中国虚拟跑步地图 — 技术方案 V4

> 编写日期：2026-07-08 | 状态：已确认  
> 版本说明：V2→V4 新增 Mobile First 设计原则 + PWA 预留

---

## 一、产品概述

一款将现实跑步数据映射到虚拟中国地图上的 **Mobile First Web 应用**。用户记录每日跑步里程，累积后在地图上沿「中国环游经典自驾路线」推进，打卡 48 个关键城市节点，解锁成就徽章。

**MVP 核心交互**：输入跑步里程 → 地图上角色沿路线前进 → 抵达城市 → 弹出打卡卡片（城市名+省份+景点介绍）。

**目标设备优先级**：手机竖屏（375px+） → iPhone / 安卓 → PC 浏览器（1920px 以内）→ 平板。

## 二、技术选型

| 层面 | 选型 | 理由 |
|------|------|------|
| 框架 | **React 19 + Vite 6** | 组件化清晰、Vite HMR 快、支持 PWA |
| 语言 | TypeScript | 类型安全，降低后期重构成本 |
| 地图 | **ECharts 中国地图**（echarts + china.js） | 零 CDN、触屏友好、散点+连线原生、中国地图合规 |
| 样式 | **手写 CSS（CSS Modules）** → 按需引入 Tailwind | MVP 手写 CSS 最小依赖，Mobile First 断点控制 |
| 状态 | **Zustand** | 轻量（~1KB），比 Redux 更适合中小项目 |
| 路由 | React Router v6 | 标准 SPA 路由 |
| 数据存储 | **localStorage**（MVP）→ IndexedDB（V2）→ 后端数据库（V3） | 渐进式升级 |
| 构建 | Vite | 开发 HMR 快，生产构建优 |
| PWA | **vite-plugin-pwa**（V2 阶段引入） | Service Worker + manifest.json |
| 部署 | CloudStudio 静态站点 | 公网可达，免费 |

## 三、Mobile First 设计原则

### 3.1 核心设计哲学

```
桌面端         →        手机端
┌──────────────────────┐
│  作为移动端的增强版    │  ← 不是各做一套，不是降级
│  而是从手机向上扩展    │
└──────────────────────┘
```

| 维度 | 移动端优先 | 桌面端增强 |
|------|-----------|-----------|
| 断点 | 375px 起，向上渐进增强 | ≥ 768px / ≥ 1024px 逐步展开 |
| 布局 | 单列瀑布流 | 多列/侧边栏 |
| 导航 | 底部 TabBar（拇指可达区） | 顶部导航/侧栏 |
| 交互 | 触控（手指区域 ≥ 44px） | 鼠标悬停/右键 |
| 地图 | 触屏手势（拖动+双指缩放） | 鼠标滚轮缩放 |

### 3.2 响应式断点策略

```css
/* 基础：375px 起（手机竖屏） */
/* >= 640px 小平板/大屏手机横屏 */
/* >= 1024px 平板横屏/桌面 */
/* >= 1440px 大屏桌面 */
```

不依赖 Tailwind 的预设断点，手写 CSS 精确控制地图容器、Tab 栏、弹窗等关键组件的响应式行为。

### 3.3 触屏交互规范

| 操作 | 支持设备 | 实现方式 |
|------|----------|----------|
| 手指拖动地图 | 触屏 | ECharts 原生手势支持 |
| 双指缩放 | 触屏 | ECharts 原生 pinch zoom |
| 点击城市节点 | 触屏+鼠标 | ECharts click 事件 |
| 横向滑动 | 触屏 | 城市卡片/统计页滑动 |
| 点击按钮/输入 | 触屏 | 手指热区 ≥ 44×44px |

### 3.4 拇指热区布局

```
┌─────────────────────┐
│     状态栏/标题       │  ← 顶部 60px（不易触及）
├─────────────────────┤
│                     │
│   地图主区域          │  ← 核心内容区
│   （全屏或近似全屏）   │
│                     │
├─────────────────────┤
│  跑步输入  │  按钮    │  ← 底部拇指热区（易触及）
├─────────────────────┤
│ 首页 │ 跑步 │ 统计 │ 成就 │  ← TabBar（拇指天然点按区）
└─────────────────────┘
```

**关键规则**：
- 主要操作按钮放在屏幕下半部分（拇指覆盖区）
- TabBar 固定底部，不跟随滚动
- 弹窗/打卡卡片从底部滑入（BottomSheet 风格，而非居中弹窗）
- 输入框聚焦时自动滚入视区

## 四、PWA 预留能力

### 4.1 当前阶段（MVP）

PWA 配置不安装、不引入，但在架构上预留：

```
frontend/public/
├── manifest.json          ← 占位文件（后续填充）
├── icons/
│   ├── icon-192x192.png   ← 占位
│   └── icon-512x512.png   ← 占位
└── sw.js                  ← 占位
```

### 4.2 V2 阶段引入

- 安装 `vite-plugin-pwa`
- 自动生成 Service Worker，静态资源离线缓存
- 生成完整 manifest.json（名称、图标、主题色、display: standalone）
- PWA 条件：HTTPS 部署（CloudStudio 原生支持）

### 4.3 预期体验

```
用户访问 → 浏览器提示"添加到主屏幕"
  → 桌面图标（类 App）
  → 全屏启动（无浏览器地址栏）
  → 离线可查看已缓存的城市数据
```

## 六、地图方案：ECharts 中国地图

### 6.1 选择理由

- **零 CDN 依赖**：echarts 和 china.js 地图数据打包进 bundle
- **触屏原生支持**：内置手势缩放（pinch）和平移（drag）
- **中国地图合规**：ECharts 官方 china.js 已正确标注
- **Mobile First 友好**：响应式容器 + 触屏事件

### 6.2 移动端地图适配

| 指标 | 手机竖屏 | PC |
|------|----------|-----|
| 地图容器高度 | 视口 60-70% | 固定 500-600px |
| 标记大小 | 8px（城市点） | 10px（城市点） |
| 标签显隐 | 仅显示当前城市名 | 全部可悬停显示 |
| 手势 | 单指拖动+双指缩放 | 鼠标拖拽+滚轮缩放 |

## 七、路线数据

- **文件**：`data/route_master_v1.json`（已冻结，参见 docs/路线冻结确认报告V1.md）
- **总长**：**21,423 km**（虚拟里程）
- **节点数**：**48 个城市**
- **比例**：跑步 1 km = 虚拟 10 km，完成全程需跑步约 **2,142 km**
- **方向**：顺时针闭环（深圳 → 全国 → 深圳）
- **难度**：普通 30 / 挑战 13 / 高原 3 / 极限 2

> ⚠️ 路线数据不做代码内联，通过 `src/data/routeLoader.ts` 加载 JSON 数据。不改变现有 `route_master_v1.json` 结构。

### 路线数据使用方式

```typescript
// src/data/routeLoader.ts
const rawData = await import('../../data/route_master_v1.json');
// 转置为前端使用的 RouteNode 类型
export const ROUTE_NODES: RouteNode[] = rawData.nodes.map(n => ({
  id: n.id,
  name: n.city,
  province: n.province,
  lng: n.longitude,
  lat: n.latitude,
  cumKm: n.total_distance_km,
  difficulty: n.difficulty,
  description: n.description,
  seaTransfer: rawData.sea_transfer?.some(s => s.route_nodes.includes(n.id))
}));
```

## 十、风险与约束

| 风险 | 影响 | 对策 |
|------|------|------|
| 手机端 ECharts 性能 | 48 个散点+路线连线可能卡顿 | 低端机降级：减少动画帧、关闭标签动画 |
| 手机键盘弹出遮挡输入 | 用户体验 | 底部输入栏 + `visualViewport` API 监听 |
| 地图触屏与页面滚动冲突 | 操作混乱 | 地图容器内手势捕获 preventDefault，外部不干扰 |
| PWA 缓存策略 | 数据版本不一致 | SW 只缓存静态资源，动态数据不走 SW |
| 路线 JSON 加载 | 打包后路径变化 | JSON 放在 `public/` 或 vite 静态导入 |
| 手机浏览器兼容 | 旧安卓 WebView 不支持 ES module | 保留 `<script nomodule>` 回退 |
