## 一、数据库总览

| 数据表 | 用途 | 核心字段 |
|--------|------|---------|
| `users` | 用户账户 | id, nickname, avatar, level, xp |
| `activities` | 运动记录 | id, user_id, distance, duration, pace, gps, source |

## 二、users 表

```sql
CREATE TABLE users (
    id              VARCHAR(64)  PRIMARY KEY,         -- 用户 ID (guest_xxx / cloud_xxx)
    nickname        VARCHAR(50)  NOT NULL DEFAULT '跑者',
    avatar          VARCHAR(20)  NOT NULL DEFAULT 'default',
    level           INT          NOT NULL DEFAULT 1,
    experience      INT          NOT NULL DEFAULT 0,
    is_guest        BOOLEAN      NOT NULL DEFAULT TRUE,
    cloud_user_id   VARCHAR(64),                      -- 云端关联 ID
    created_at      TIMESTAMP    NOT NULL DEFAULT NOW(),
    last_sync_at    TIMESTAMP,
    INDEX idx_cloud_user_id (cloud_user_id)
);
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | VARCHAR(64) | 主键，游客为 `guest_xxx`，登录后为 `cloud_xxx` |
| `nickname` | VARCHAR(50) | 用户昵称 |
| `avatar` | VARCHAR(20) | 头像 key，如 'default'/'runner' |
| `level` | INT | 等级（1-12） |
| `experience` | INT | 总经验值（成就奖励累计） |
| `is_guest` | BOOLEAN | 是否为游客（true=离线模式） |
| `cloud_user_id` | VARCHAR(64) | 游客绑定云端后的正式 ID |
| `created_at` | TIMESTAMP | 账户创建时间 |
| `last_sync_at` | TIMESTAMP | 最近一次同步时间 |

## 三、activities 表

```sql
CREATE TABLE activities (
    id              VARCHAR(64)  PRIMARY KEY,         -- 记录 ID (run_timestamp_random)
    user_id         VARCHAR(64)  NOT NULL,             -- 用户 ID
    source          VARCHAR(20)  NOT NULL DEFAULT 'manual',
    activity_type   VARCHAR(20)  NOT NULL DEFAULT 'running',
    distance_km     DECIMAL(8,2) NOT NULL,             -- 距离（公里）
    duration_sec    INT          NOT NULL,             -- 时长（秒）
    pace_sec        DECIMAL(6,2) NOT NULL,             -- 配速（秒/公里）
    calories        INT,                               -- 消耗热量
    start_time      TIMESTAMP    NOT NULL,             -- 运动开始时间
    gps_track       JSON,                              -- GPS 轨迹点数组
    note            TEXT,
    created_at      TIMESTAMP    NOT NULL DEFAULT NOW(),

    INDEX idx_user_date (user_id, start_time DESC),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 关键设计

- **`distance_km`**：单条记录的真实距离。**所有统计均由 SUM 计算，不允许单独存 total 字段**
- **`gps_track`**：JSON 字段存坐标点数组 `[{lat, lng, timestamp, altitude, speed}]`
- **索引**：`(user_id, start_time DESC)` 覆盖查询用户按时间倒叙的所有记录

## 四、achievements 表

```sql
CREATE TABLE achievements (
    id              VARCHAR(32)  PRIMARY KEY,         -- 成就 ID (dist_10 / city_1 / streak_7)
    name            VARCHAR(50)  NOT NULL,
    description     TEXT,
    icon            VARCHAR(10)  NOT NULL,            -- emoji
    category        VARCHAR(20)  NOT NULL,            -- distance / city / streak / special
    condition_type  VARCHAR(20)  NOT NULL,            -- total_km / city_unlock / streak_days
    threshold       DECIMAL(10,2) NOT NULL,            -- 阈值
    xp_reward       INT          NOT NULL DEFAULT 0
);
```

---

## 五、user_achievements 表

```sql
CREATE TABLE user_achievements (
    user_id         VARCHAR(64)  NOT NULL,
    achievement_id  VARCHAR(32)  NOT NULL,
    progress        DECIMAL(5,4) NOT NULL DEFAULT 0,   -- 0.0000 ~ 1.0000
    unlocked        BOOLEAN      NOT NULL DEFAULT FALSE,
    unlocked_at     TIMESTAMP,

    PRIMARY KEY (user_id, achievement_id),
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (achievement_id) REFERENCES achievements(id)
);
```

---

## 六、city_unlocks 表

```sql
CREATE TABLE city_unlocks (
    id              SERIAL       PRIMARY KEY,
    user_id         VARCHAR(64)  NOT NULL,
    city_id         VARCHAR(32)  NOT NULL,            -- 城市节点 id (shenzhen / xiamen / ...)
    city_order      INT          NOT NULL,            -- 顺序编号 1-48
    city_name       VARCHAR(20)  NOT NULL,
    reached_at      TIMESTAMP    NOT NULL DEFAULT NOW(),

    UNIQUE KEY (user_id, city_id),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

---

## 七、sync_logs 表

```sql
CREATE TABLE sync_logs (
    id              SERIAL       PRIMARY KEY,
    user_id         VARCHAR(64)  NOT NULL,
    sync_type       VARCHAR(20)  NOT NULL,            -- 'upload' / 'download' / 'full'
    status          VARCHAR(20)  NOT NULL,            -- 'success' / 'error'
    records_uploaded INT         DEFAULT 0,
    records_downloaded INT       DEFAULT 0,
    error_message   TEXT,
    created_at      TIMESTAMP    NOT NULL DEFAULT NOW(),

    INDEX idx_user_sync (user_id, created_at),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## 八、ER 关系图

```
users (1) ──→ (N) activities
users (1) ──→ (N) user_achievements ──→ (1) achievements
users (1) ──→ (N) city_unlocks
users (1) ──→ (N) sync_logs

数据一致性规则：
  activities.distance_km SUM = users 的 "个人累计跑量"
  activities 全量 SUM × 1:10 = 虚拟累计公里
  所有用户的 activities SUM = 全民累计跑量（禁止单独存 total 字段）
```
