## 三、代码审查发现的问题（2 个，均已修复）

### 问题 1：首屏深圳城市未自动解锁

**描述**：`cityStore.initialize()` 仅从 localStorage 读取，首次使用时为空列表。`checkAndUnlock` 的 `totalDistanceKm > 0` 守卫导致深圳只能在首次添加跑量后才解锁。

**影响**：低。用户首次打开看不到已解锁的起点城市。

**修复**：在 `cityStore.initialize()` 中，如果无已存数据，自动创建深圳起点记录并写入 localStorage。

### 问题 2：CityBottomSheet 关闭时潜在的 null 访问

**描述**：`handleDismiss` 延迟 300ms 调用 `dismissAlert()`，但 React 在中间状态中 `alertCity` 仍为真，导致 `city.difficulty` 访问空值风险。

**影响**：低。300ms 窗口内触发可能导致 JS 异常。

**修复**：移除延迟，`handleDismiss()` 立即调用 `dismissAlert()`，依靠 CSS transition 处理关闭动画。
