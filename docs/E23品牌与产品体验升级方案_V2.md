# E23跑起来 — 品牌与产品体验升级方案 V2

| 问题 | 根因 | 严重度 |
|------|------|--------|
| 安卓图标未变化 | 旧构建缓存 + build.gradle namespace 未更新 + capacitor config appId 不一致 | P0 |
| 验证码收不到 | 无真实短信服务，仅 mock，前端未诚实展示测试码 | P0 |

## 2. 品牌定位

- **产品定位**：E23 同学的团队跑步与环游挑战平台
- **用户价值**：记录每一次出发，汇聚团队里程，一起完成环游旅程
- **核心口号**：一起出发，一直向前
- **产品语气**：真诚、坚定、有团队感、有戈壁精神、不过度煽情
- **视觉关键词**：戈壁暖色、深蓝夜空、暖橙沙丘、白色文字、简约现代

## 3. 视觉策略

- **主色**：`#0D2B45`（深蓝夜空，背景大面积使用）
- **辅色**：`#F28C22`（暖橙戈壁，强调/CTA用）
- **中性色**：`#FAD7A0`、`#FFF4E0`（沙丘过渡）
- **文字**：`#FFFFFF`（主）/ `#FAD7A0`（辅助）/ `#8899AA`（弱化）
- **图标系统**：纯 CSS 矢量图标 + 内联 SVG 组件，统一风格、统一直径
- **地图风格**：深色底图 + 金色路线 + 暖橙地标

## 4. 信息架构（首页重设计）

新首页布局（从上到下）：
```
[状态栏区]        — 问候语 + 当前城市
[核心数据区]      — 今日跑量 / 累计 / 虚拟里程 / 连续天数 (2×2 Grid)
[进度卡片]        — E23当前抵达: 城市名 → 下一站 → 进度条
[地图区]          — 中国环游地图（压缩高度）
[最近运动]        — 最近一条跑步记录摘要
[开始跑步]        — 显性CTA按钮
```

## 5. 图标策略

- 放弃所有 emoji 导航图标
- 使用内联 SVG 组件（`E23Icon` 统一组件）
- 4 个 Tab：首页/跑步/排行/我的 → 简约几何风格
- Android 图标：方案四（沙丘+E23+跑者剪影）已就绪

## 6. 登录与验证码策略

- **当前状态**：无真实短信服务，只有 mock
- **落地方案**：显示测试验证码 + 60s 倒计时 + 明确标记"测试版"
- **不允许**：假装短信已发送
- **后续**：接入阿里云/腾讯云短信需要后端部署 + 密钥配置

## 7. 技术落地计划

1. 修正 build.gradle（appId/version）
2. 构建新 APK + 签名验证
3. 重写首页（MainApp.tsx + App.css）
4. 统一导航图标
5. 添加空状态/引导
6. 登录页文案修正
7. 全量构建 + 验证

## 8. 验收标准

- 桌面图标显示 E23跑起来新图标
- 桌面名称：E23跑起来
- 启动页：E23跑起来 + 从戈壁出发，奔向世界
- 登录页：测试验证码明确显示
- 首页：信息层级清晰，核心行动明确
- 导航：使用统一图标
- APK：签名通过，可安装
