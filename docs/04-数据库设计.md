## 二、实体关系图

```
User (用户) ── 1:1 ── UserStats (运动统计)
  │
  ├── 1:N ── RunRecord (跑步记录)
  │
  ├── 1:N ── UnlockedCity (已解锁城市)
  │
  └── 1:N ── UnlockedAchievement (已解锁成就)

Route (路线) ── 1:N ── RouteNode (路线节点)
  └── 1:N ── Milestone (里程碑节点)

Achievement (成就定义) ── 1:1 ── AchievementCondition (解锁条件)
City (城市定义)
```

## 三、核心数据模型

### 3.1 RouteNode（路线节点 — 静态数据）

以 TypeScript 常量数组定义，是系统的核心数据源。

```typescript
interface RouteNode {
  id: string;                        // 城市唯一标识，如 "beijing"
  order: number;                     // 路线顺序号，1-48
```

### 3.4 RunRecord（跑步记录 — 用户数据）

```typescript
interface RunRecord {
  id: string;                        // 如 "run_20260708_001"
```

## 四、localStorage 存储结构（MVP 实际格式）

```typescript
// localStorage key: "vr_china_v1"
interface AppData {
  version: number;              // 数据版本号，迁移时递增
  settings: UserSettings;
  stats: UserStats;
  runRecords: RunRecord[];
  unlockedCities: string[];     // 已解锁城市的 cityId 列表
  unlockedAchievements: string[]; // 已解锁成就的 achievementId 列表
}

// 初始值
const DEFAULT_DATA: AppData = {
  version: 1,
  settings: {
    nickname: "跑步达人",
    scaleRatio: 10
  },
  stats: {
    totalDistance: 0,
    totalRuns: 0,
    currentStreak: 0,
    longestStreak: 0,
    lastRunDate: null,
    unlockedCityCount: 0,
    totalCities: 48
  },
  runRecords: [],
  unlockedCities: [],
  unlockedAchievements: []
};
```

**容量预估（单用户完整数据）**：
| 数据项 | 大小 |
|--------|------|
| settings | ~100B |
| stats | ~200B |
| runRecords（200 条） | ~10KB |
| unlockedCities（48 个） | ~2KB |
| unlockedAchievements（20 个） | ~1KB |
| **总计** | **~13KB**（远低于 5MB 限制） |

## 五、数据版本迁移

当数据结构变更时（如新增字段），版本号递增，读取时自动迁移：

```typescript
function migrateData(data: AppData): AppData {
  if (data.version === 1) {
    // v1 → v2: 新增字段示例
    // data.someNewField = defaultValue;
    data.version = 2;
  }
  // ... 链式迁移
  return data;
}
```

## 七、V3 多用户表设计（预告）

```
users
├── id (UUID, PK)
├── nickname
├── avatar
├── created_at
└── updated_at

run_records
├── id (UUID, PK)
├── user_id (FK → users.id)
├── date (DATE)
├── distance (FLOAT)
├── note (TEXT)
└── created_at (TIMESTAMP)

unlocked_cities
├── id (UUID, PK)
├── user_id (FK → users.id)
├── city_id (VARCHAR)
└── unlocked_at (TIMESTAMP)

unlocked_achievements
├── id (UUID, PK)
├── user_id (FK → users.id)
├── achievement_id (VARCHAR)
├── notified (BOOLEAN)
└── unlocked_at (TIMESTAMP)
```
