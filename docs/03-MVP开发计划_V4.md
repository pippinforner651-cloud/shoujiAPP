## 一、MVP 目标

**一句话**：用 React + Vite + TypeScript 搭建一个 **Mobile First** Web 应用，用户输入跑步里程后，在 ECharts 中国地图上沿 48 城自驾路线推进，城市打卡 + 成就解锁。

**目标设备优先级**：手机竖屏 (375px+) → iPhone / 安卓 → PC 浏览器 (1920px) → 平板

**必须交付（7 项）**：
1. ✅ 用户昵称设置
2. ✅ 本地 localStorage 数据持久化
3. ✅ 输入每日跑量（触屏友好）
4. ✅ 自动计算累计跑量
5. ✅ 地图路线推进 + 触屏交互（ECharts 中国地图 + 路线连线 + 跑者标记）
6. ✅ 当前到达城市显示（底部滑入式城市打卡卡片 BottomSheet）
7. ✅ 个人统计页面

**暂不开发**：
- ❌ GPS 自动同步
- ❌ 微信登录 / OAuth
- ❌ 支付 / 商业化
- ❌ 后端 API
- ❌ 好友 / 排行榜
- ❌ PWA 离线缓存（V2 引入）

**Mobile First 原则贯穿所有 Phase**（不单独做"移动端适配"阶段）：

| 阶段 | 移动端关注点 |
|------|-------------|
| Phase 0 | 目录预留、viewport 设置、占位图标 |
| Phase 1 | 地图容器响应式、触屏手势、拇指热区 |
| Phase 2 | 底部输入栏、BottomSheet 弹窗、键盘适配 |
| Phase 3 | 统计卡片响应式布局 |
| Phase 4 | 成就墙网格自适应 |
| Phase 5 | 构建优化、PWA 占位验收 |

## 五、路线数据确认

- **数据源**：`data/route_master_v1.json`（已冻结 V1.0）
- **总长**：21,423 km（虚拟里程）
- **城市**：48 城，深圳→深圳顺时针闭环
- **比例**：1:10，完成需跑约 2,142 km
- **难度**：普通 30 / 挑战 13 / 高原 3 / 极限 2
- **路线数据不做代码内联**，通过 `src/data/routeLoader.ts` 加载 JSON
- **sea_transfer 类型**保留在数据中，前端展示时特殊标注轮渡段
