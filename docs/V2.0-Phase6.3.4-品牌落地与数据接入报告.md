## 3. 数据接入结果（按数据接入真相表）

### 3.1 本阶段真实接通

| 数据源 | 状态 | 实际能力 |
|--------|------|----------|
| **APP 内 GPS 跑步** | ✅ **已真实接通** | `RunSession` 组件使用 `navigator.geolocation.watchPosition`，Android 端有 GPS 权限，**已在 Android 端运行** |
| **手动录入** | ✅ **已真实接通** | `ManualAdapter` + 真实表单输入，进入 runStore |
| **GPX 文件导入** | ✅ **已真实接通** | `FileImportAdapter` 解析 GPX XML，进入 runStore |

### 3.2 安卓原生推进中

| 数据源 | 状态 | 卡点 |
|--------|------|------|
| **Health Connect** | ⚠️ **部分完成 / 未接通** | 接口设计完成（`HealthConnectAdapter.ts`），但 PWA 浏览器环境无法调用 Health Connect API。需要在原生 Android 壳里用 Kotlin + Health Connect 客户端读取并通过 Capacitor Plugin 暴露给前端。**当前为 code-only / 未真机授权验证** |

### 3.3 仅接口预留（未真实接通）

| 数据源 | 真实状态 |
|--------|----------|
| **Apple HealthKit** | 🔴 **未接通**。PWA 完全无法读取 HKHealthStore。需 iOS 原生 APP + Swift + Apple Developer 账号（¥688/年） |
| **COROS 高驰** | 🔴 **未取得权限**。CorosAdapter 仅含 OAuth 流程设计。未获得 COROS 官方 API 授权前不得写"已接通" |
| **悦跑圈** | 🔴 **核验中**。JoyrunAdapter 仅含流程设计。开放平台是否仍接受新应用待核验 |
| **微信运动** | 🔴 **未接通**。微信登录仅用于身份认证。微信运动跑步数据无公开 API，禁止爬虫/抓包/模拟登录 |
| **华为运动健康** | 🔴 **未接通**。华为 Health Kit 需华为开发者联盟账号注册应用，未完成 |

### 3.4 数据接入真相表

| 数据源 | 状态 | 当前接入位置 | 限制 |
|--------|------|--------------|------|
| APP 内 GPS | ✅ 真实接通 | `services/activitySources/AppGpsAdapter.ts` | — |
| 手动录入 | ✅ 真实接通 | `services/activitySources/ManualAdapter.ts` | — |
| GPX 导入 | ✅ 真实接通 | `services/activitySources/FileImportAdapter.ts` | — |
| Health Connect | ⚠️ 接口代码就绪 | `services/activitySources/HealthConnectAdapter.ts` | PWA 不可用，需 Android 原生 + Capacitor 插件 |
| HealthKit | 🔴 接口代码就绪 | `services/activitySources/HealthKitAdapter.ts` | PWA 不可用，需 iOS 原生 + Swift |
| COROS | 🔴 接口代码就绪 | `services/activitySources/CorosAdapter.ts` | 待官方 API 授权 |
| 悦跑圈 | 🔴 接口代码就绪 | `services/activitySources/JoyrunAdapter.ts` | 待开放平台核验 |
| 微信运动 | 🔴 未接通 | — | 无公开 API，禁止非官方方式 |

### 3.5 统一数据模型（已升级 Phase 6.3.2）

- `id`, `userId`, `source` (9 种枚举), `sportType`
- `distanceMeters`, `durationSeconds`, `paceSecondsPerKm`
- `calories`, `avgHeartRate`, `maxHeartRate`, `elevationGain`
- `routeData`, `deviceName`, `syncTime`
- `verificationStatus` (6 级)
- `rawDataHash` (去重)

### 3.6 Health Connect 攻关详情

**当前代码就绪度：**
- `HealthConnectAdapter` 实现了完整接口（connect / getAuthorizationStatus / syncActivities / normalizeActivity / getLastSyncTime）
- 包含字段映射注释（距离/时长/心率/卡路里/轨迹）
- 文档中说明所需权限（android.permission.health.READ_STEPS、READ_DISTANCE、READ_HEART_RATE 等）

**真机验证路径（待执行）：**
1. 在 Android 13+ 手机安装 Android Studio
2. 使用 `@capacitor/android` 在原生壳中实现 Health Connect 桥接
3. 编写 Kotlin 插件 `com.e23running.healthconnect`
4. 用 `HealthConnectClient` 读取 `ExerciseSessionRecord`
5. 通过 Capacitor Bridge 返回给前端
6. 在 RunSession 完成时调用 `connect()` 拉取历史
7. 通过 `syncActivities()` 增量同步

**当前卡点：**
- 用户尚未在真机授权
- 本地无 Android Studio 环境
- 项目未原生化到 Kotlin 模块
