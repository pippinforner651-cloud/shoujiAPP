## 二、OSM 数据来源

### 2.1 OpenStreetMap

- **数据源**：[OpenStreetMap](https://www.openstreetmap.org) 全球路网数据
- **中国覆盖**：包含中国全境高速公路、国道、省道、县道
- **更新频率**：社区持续更新
- **数据属性**：每条道路包含 speed limit、车道数、路面类型等标签（OSM tags）
- **导出格式**：OSM XML、PBF（Protocol Buffer Binary）

### 2.2 与 route_master_v1.json 的对比

| 维度 | route_master_v1.json | OSM + OSRM |
|------|---------------------|------------|
| 数据性质 | 人工估算的直线距离 | 真实道路路径 |
| 深圳→厦门 | 570 km | 561.1 km |
| 路线形状 | 直线（A→B 插值） | 沿 G15 沈海高速弯曲 |
| 海拔数据 | 无 | 可获取（OSM 含 elevation）|
| 路线复杂度 | 48 段直线 | 数千个道路坐标点 |

> **结论**：OSM 数据可用作真实路线几何，与现有里程数据偏差 < 2%，可安全替换。

## 三、路由引擎选择

### 3.1 方案对比：OSRM vs GraphHopper

| 维度 | OSRM | GraphHopper |
|------|------|-------------|
| **开源协议** | `MIT` | `Apache 2.0` |
| **公共 API** | `router.project-osrm.org` 无密钥 | 需要 API key |
| **免费额度** | 无限（公共 demo） | 500-1000 req/天 |
| **GeoJSON 输出** | ✅ `geometries=geojson` | ✅ `points_encoded=false` |
| **简化几何** | ✅ `overview=simplified`（51pts/段） | 自动简化 |
| **响应速度** | ~1-2s/段 | ~1-2s/段 |
| **自部署难度** | 中等（C++，需要编译） | 较易（Java，Spring Boot） |
| **中国路网** | 需中国 OSM 地图数据 | 需中国 OSM 地图数据 |

### 3.2 推荐：OSRM（验证阶段）

**理由**：
1. 无需 API key，公共 demo 可直接使用
2. `geometries=geojson` 输出坐标格式与 ECharts 完全兼容
3. `overview=simplified` 将每段压缩到 ~50 个坐标点（1 KB/段）
4. 48 段合计 ≈ 50 KB，前端加载无压力

### 3.3 生产环境建议

如果需要自部署路由引擎确保可靠性：
- OSRM 自部署：`docker run -t -v $(pwd):/data osrm/osrm-backend osrm-routed --algorithm mld /data/china-latest.osrm`
- GraphHopper 自部署：`docker run -d -p 8989:8989 -v $(pwd)/data:/data graphhopper/graphhopper:8.0`

## 四、数据生成流程

### 4.1 整体流程

```
┌─────────────┐    ┌───────────────┐    ┌───────────────────┐
│ route_master  │───→│ OSRM API 批量  │───→│ route_geometry_v1  │
│ _v1.json      │    │ 路由请求      │    │ .json             │
│ (48 城坐标)   │    │               │    │ (完整几何数据)    │
└─────────────┘    └───────────────┘    └───────────────────┘
                          │                      │
                          ▼                      ▼
                 每段请求耗时 ~1-2s      最终产出 ~50 KB
                 48 段 + 闭环 = 49 段   包含所有路段 GeoJSON
                 总计 ~60-90s
```

### 4.2 OSRM 请求格式（示例）

```
GET https://router.project-osrm.org/route/v1/driving/
    114.07,22.54;118.08,24.48           ← 深圳→厦门坐标
    ?overview=simplified                ← 简化几何
    &geometries=geojson                 ← GeoJSON 输出
    &steps=false                        ← 不需要导航指令
```

### 4.3 批量生成脚本（伪代码）

```python
import requests, json

# 读取城市节点坐标
route = json.load(open('data/route_master_v1.json'))
nodes = route['nodes']

geometry = {
  "route_version": "1.0",
  "geometry_version": "1.0",
  "source": "OSRM router.project-osrm.org",
  "profile": "driving",
  "overview": "simplified",
  "created_at": "2026-07-09",
  "segments": []
}

# 遍历每段城市对
for i in range(len(nodes)):
    from_node = nodes[i]
    to_node = nodes[(i + 1) % len(nodes)]  # 闭合：最后一段回深圳
    
    coords = f"{from_node.longitude},{from_node.latitude};{to_node.longitude},{to_node.latitude}"
    url = f"https://router.project-osrm.org/route/v1/driving/{coords}?overview=simplified&geometries=geojson&steps=false"
    
    resp = requests.get(url).json()
    route = resp['routes'][0]
    
    geometry['segments'].append({
        "from": from_node.id,
        "from_city": from_node.city,
        "to": to_node.id,
        "to_city": to_node.city,
        "distance_km": round(route['distance'] / 1000, 1),
        "duration_min": round(route['duration'] / 60, 1),
        "geometry": route['geometry']  # GeoJSON LineString
    })

json.dump(geometry, open('data/route_geometry_v1.json', 'w'))
```

### 4.4 请求注意事项

| 注意事项 | 说明 |
|---------|------|
| 请求间隔 | 建议间隔 0.5-1s，避免公共 API 限流 |
| 失败重试 | 添加 3 次重试 + 指数退避 |
| 耗时代估 | 49 段 × 1.5s ≈ 75s |
| 缓存策略 | 生成后 commit 到仓库，不重复请求 |

## 六、如何接入 ECharts

### 6.1 前端改动范围

仅需修改 `ChinaMap/option.ts` 中的 `buildRouteLineSeries()` 函数：

**当前（直线）**：
```javascript
// 使用城市间直线坐标
coords = [[shenzhen.lng, shenzhen.lat], [xiamen.lng, xiamen.lat], ...]
```

**改为（真实道路）**：
```javascript
// 从 route_geometry_v1.json 加载道路几何
import routeGeometry from '../../data/route_geometry_v1.json';

// 合并所有段的 coordinates 为一条连续路线
const roadCoords = routeGeometry.segments
  .flatMap(seg => seg.geometry.coordinates);

// 传入 lines series
data: [{ coords: roadCoords }]
```

### 6.2 核心变更文件

| 文件 | 变更 |
|------|------|
| `data/route_geometry_v1.json` | 🆕 新增（OSRM 生成） |
| `src/data/roadGeometryLoader.ts` | 🆕 新增（加载 + 合并几何数据） |
| `src/utils/routeProgress.ts` | 🖊️ 改为基于道路几何的插值算法 |
| `src/components/ChinaMap/option.ts` | 🖊️ 路线系列从直线坐标改为道路坐标 |
| `src/components/RunnerMarker/calcPosition.ts` | 🖊️ 跑者位置沿道路几何定位 |

### 6.3 跑者定位算法升级

**当前**：在城市两点之间做线性插值（直线路径）

**升级后**：沿道路几何点序列进行分段插值
```
1. 计算每段道路几何的长度（sqrt(dlng² + dlat²)）
2. 找到当前 virtualKm 落在哪个道路点之间
3. 对该道路 segment 做线性插值
4. 输出精确到道路上的坐标
```

## 七、风险与注意事项

| 风险 | 等级 | 说明 | 对策 |
|------|------|------|------|
| OSRM 公共 API 限流 | 🟡 中等 | 48 段连续请求可能被短暂封禁 | 间隔 1s 请求 + 3 次重试 |
| OSM 中国路网完整性 | 🟡 中等 | 偏远地区（西藏阿里、新疆喀什）路网可能不完整 | 验证后补全，若有缺失段回退到直线 |
| `overview=simplified` 精度 | 🟢 低 | 简化后可能偏离实际道路 | 目测验证，必要时对关键路段用 `full` |
| 闭环段（广州→深圳） | 🟢 低 | 珠江口绕行虎门大桥 vs 深中通道 | OSRM 自动计算最优路径 |
| 海渡段（北海→海口） | 🟡 中等 | 轮渡段 OSRM 无法计算（无道路） | 保留现有 sea_transfer 直线段 |
| JSON 体积增长 | 🟢 低 | 55 KB vs 当前 0 KB | 影响极小，gzip 后可忽略 |
| 与现有里程差异 | 🟢 低 | 每段偏差 < 2% | 以 OSRM 实际距离为准，不影响用户体验 |
