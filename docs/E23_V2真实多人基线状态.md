# E23 V2真实多人基线状态

更新时间：2026-07-15

## 已完成

- 后端依赖锁文件、Prisma客户端与当前Schema恢复一致；
- 后端历史活动字段统一为`distanceMeters`和`durationSeconds`，旧接口展示仍按公里转换；
- 新增`Community`、`CommunityMember`、`RoutePackage`、`RelayEvent`和`ActivityContribution`；
- 原始运动与活动贡献分离；
- 活动来源按用户、来源和来源活动ID去重；
- 同一运动在同一活动中由`eventId + activityId`唯一约束防止重复累计；
- 贡献归属由服务端查询`Activity.userId`派生，客户端提交`userId`会被拒绝；
- 写入前校验活动状态与时间窗口、E23成员资格、运动审核状态和既有贡献；
- 排行榜聚合归属读取运动所有者，不信任贡献行中的冗余用户字段；
- 已审核成员的设备/平台运动按1:1自动计入；
- 手工记录待审核，未审核成员、疑似重复和无效记录不计入；
- 班级进度、排行、封顶和超额贡献纯函数及API契约完成；
- V2 HTTP路由工厂完成，未知活动返回404；
- 前端V2进度DTO映射完成；
- V1多人界面恢复读取现有真实排行榜，并保留冻结的1:10计算；
- 多人模式明确区分`v1-backend`、`v2-backend`和`disabled`，V2没有显式活动ID时保持关闭；
- 请求失败显示“数据加载失败”，服务未启用显示“多人服务暂未启用”，不再生成模拟跑者、虚假排行或伪装成0人0公里；
- 后端模拟微信openid和token已移除，未配置时返回`WECHAT_NOT_CONFIGURED`。
- 后端固定短信验证码和模拟token已停用，未配置正式短信服务时返回`PHONE_AUTH_NOT_CONFIGURED`；
- 旧模拟跑者与短信适配器文件当前没有生产调用方，不参与正式进度、排行或身份数据。
